## 3. Screen Implementation: `app/(tabs)/profile.tsx`

Implement the profile administration interface. Ensure **every interaction pressable** initiates a distinct `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light)` buzz response.

### A. Authentication Core & State Hooks
* Extract `isLoaded`, `isSignedIn`, and `signOut` natively from Clerk's `useAuth()`.
* Extract `user` properties via Clerk's `useUser()`.

### B. Conditional UI Rendering Strategy

#### State 1: User Is Authenticated (`isSignedIn === true`)
Render a clean card container block with the following elements:
* **User Profile Image:** Display a large avatar image using `user.imageUrl`. If it is undefined or fails to parse, fall back to displaying the first letter of their first name inside a stylized placeholder circle.
* **User Metadata details:** Render the user's full name and primary email address strings inside clear text labels.
* **Action Button:** Include a wide, prominent rectangular `Pressable` button styled in a deep **Red** color layout with bold **White** typography displaying `"Sign Out"`.
    * **Action Logic:** On press, trigger a haptic pulse, call `signOut()`, and clear the local state layout context cleanly.

#### State 2: User Is Guest / Not Authenticated (`isSignedIn === false`)
Render a simplified call-to-action placeholder within the card container:
* **Avatar Placeholder:** Render an empty/generic grayscale human silhouette vector icon.
* **Metadata Message:** Clear the name and email fields entirely, displaying an instructional header saying `"Not Logged In"`.
* **Action Redirect Box:** Make the entire placeholder card block a `Pressable` target wrapper.
    * **Action Logic:** When clicked, fire a tactile haptic pulse and cleanly route the user directly over to the main login workspace using `router.push('/sign-in')`.

## 1. Prerequisites & Dependency Check
Before generating or installing the components, verify that the following dependencies are installed or instruct the user to install them if missing:
* `clerk-expo` or `@clerk/clerk-expo`
* `expo-router`
* `lucide-react-native`
* `expo-haptics`
* `react-native-safe-area-context`

---

## 2. File Architecture & Implementation

### File 1: `app/_layout.tsx`
Create the root layout file. It must include the following:
* Configure the `<Stack/>` navigator.
* Set `screenOptions` to hide the default header (`headerShown: false`).
* Set the `contentStyle` background color to `#fff8ef`.
* Integrate a `<StatusBar/>` component matching the theme.

### File 2: `app/index.tsx`
Create the main landing screen using styles/tokens inspired by a `themes.ts` file (assume standard StyleSheet variables for clean layout).
* Wrap the screen in a `SafeAreaView` and a main `View` container.
* **Top Bar Layout:** Create a header row split into two sections:
    * **Top-Left Section:** A horizontal `View` containing the `ChefHat` icon from `lucide-react-native` and the text **"TasteMate"** styled in a clean, small font.
    * **Top-Right Section:** An authentication status area.
* **Authentication Logic:**
    * Import `useAuth` and `useUser` from Clerk.
    * Extract `isLoaded` and `isSignedIn` from `useAuth`, and `user` from `useUser`.
    * Import `useRouter` from `expo-router`.
* **Conditional Auth UI:**
    * If the user **is NOT** signed in (`!isSignedIn`), display a "Sign In" button with an accompanying icon.
    * Implement a `goToSignIn` function triggered by the button's `onPress`.
    * Trigger an `expo-haptics` feedback effect (e.g., impact light/medium) upon pressing the button before navigating to the sign-in screen.

### File 3: `app/(auth)/sign-in.tsx`
Create a new folder named `(auth)` inside the `app` directory, and place a `sign-in.tsx` file inside it.
* Wrap the screen structure in a `SafeAreaView`, followed by a main container `View`.
* Include a `Pressable` back button.
* Implement a `handleBack` logic function that routes the user back to the index page (`app/index.tsx`).
* Provide a clean implementation of the Clerk Sign-In flow, ensuring successful authentication seamlessly routes the user back to the main page.

## 3. Design Tokens Reference (Themes)
* **Background Color:** `#fff8ef`
* **Typography:** Small, clean, modern branding for "TasteMate".
* **Layout:** Space-between alignment for the header row to perfectly separate the logo from the auth button.

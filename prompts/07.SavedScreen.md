# Task: Implement Saved Meals Screen with Batch Deletion & Multi-Select

Implement the Saved Meals management screen featuring dynamic header states, single-item deletion workflows, and an advanced long-press multi-select batch deletion interface.

## 1. Environment & State Initialization
**File:** `(tabs)/saved.tsx`

### A. Core Hooks & Auth Context
*   **Authentication Hooks:** 
    *   Extract `isLoaded` and `isSignedIn` from Clerk's `useAuth()`.
    *   Extract `isAuthenticated` from Convex's `useConvexAuth()`.
*   **Navigation:** Import `router` from `expo-router`.
*   **Global Actions:** Import `setPendingAuthAction` from `store/useAppStore.ts`.
*   **Convex API Integration:**
    *   Bind the `getSavedMeals` query.
    *   Bind the `deleteMeal` mutation.

### B. UI Component States
Initialize the following React state hooks:
*   `deletingId`: `string | null` (Tracks an active single-item deletion network call).
*   `isMultiSelectMode`: `boolean` (Toggles the batch selection overlay/footer view).
*   `selectedMealIds`: `Set<string>` or `string[]` (Tracks checked IDs during multi-select).

### C. Navigation Cleanup Lifecycle
*   Implement a `useEffect` hook that listens to component unmount boundaries or router path shifts. If the user navigates away from this screen, reset `isMultiSelectMode` to `false` and clear `selectedMealIds`.

---

## 2. Header Architecture
Render a sticky header layout wrapping these elements:
*   **Top-Left Asset:** A pressable back arrow icon (`"<"`) accompanied by the text "Back". Triggers `router.back()`.
*   **Top-Right Asset:** 
    *   Conditional Render: If `isSignedIn` is true, render a User ID Avatar icon. Clicking this redirects the user via `router.push("/profile")`.
    *   Else, render a standard "Sign In" text link/button. Clicking this sets `setPendingAuthAction("saved-page")` and pushes to the login gateway.

---

## 3. Screen Body & Layout Logic (Inside `<ScrollView>`)

Handle rendering hierarchy via strict conditional evaluations:

### State A: Loading Barrier
*   If `savedMeals === undefined`, render a centralized loading indicator accompanied by the text `"Loading..."`.

### State B: Empty Set Boundary
*   If `savedMeals.length === 0`, hide listings and render:
    *   The centered message: `"No saved meals yet"`.
    *   A primary CTA button titled `"Discover"` outfitted with a compass icon. Pressing this executes a route switch to the Discover tab.

### State C: Active Dataset Grid (Meals Mapping)
Map over the `savedMeals` collection. Match the structural visual styling rules found in the discover view (cards with explicit outer margins, subtle shadows, and rounded borders).

*   **Card Framework:** Split vertically:
    *   **Upper Half:** Card Image (`imageUrl`).
    *   **Lower Half:** Render `category` and the bolded meal `name`.
*   **Interaction Handlers:**
    *   `onLongPress`: If `isMultiSelectMode` is false, toggle `isMultiSelectMode = true` and insert this card's ID into `selectedMealIds`.
    *   `onPress`: 
        *   If `isMultiSelectMode` is true, toggle this card's ID inclusion inside the `selectedMealIds` set.
        *   If `isMultiSelectMode` is false, route to the detailed preview of this meal.

---

## 4. Deletion Workflows

### Workflow 1: Single Card Deletion Button
Position a concrete, rounded button near the bottom edge of the card (mirrored from the discover layout style, respecting a margin of `2`).
*   **Style:** Deep red container background, high-contrast white text labeled `"Remove"`, alongside a trashcan icon.
*   **Disabling Rules:** Force button disabled state if `isMultiSelectMode === true` OR if `deletingId === currentMeal._id`.
*   **Text Mutation:** If `deletingId === currentMeal._id`, dynamically alter the string payload to read `"Deleting..."`.
*   **Execution Handler (`handleDelete`):**
    1.  Present a native validation prompt (`Alert.alert`) confirming removal action ("Cancel" vs "Remove").
    2.  Upon confirmation, set `deletingId` to the target ID.
    3.  Execute `await deleteMeal({ id })`.
    4.  Wrap in a `finally` block resetting `deletingId` to `null`.

### Workflow 2: Bottom Sticky Batch Deletion Menu
When `isMultiSelectMode` is active, display a floating global action footer at the bottom of the viewport viewport layout.
*   **Layout:** Houses a global red trashcan action icon alongside a count indicator showing how many items are checked.
*   **Execution Handler (`handleBatchDelete`):**
    1.  Trigger an native confirm popover: `"Are you sure you want to delete {number} items?"` with options to Cancel or Remove.
    2.  Upon confirmation, block UI actions and iterate concurrent promises: `await Promise.all(Array.from(selectedMealIds).map(id => deleteMeal({ id })))`.
    3.  Clear state vectors upon resolution: Reset `selectedMealIds` to an empty array and toggle `isMultiSelectMode` to `false`.

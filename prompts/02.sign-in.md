1. Context & Architecture Requirements
File Location: app/(auth)/sign-in.tsx

Dependencies Used: @clerk/clerk-expo, expo-web-browser, expo-haptics, lucide-react-native, react-native-safe-area-context. linear-gradient

Deep Linking Rule: Rely strictly on the exported redirect configuration from ../../lib/auth which uses AuthSession.makeRedirectUri({ scheme: "mealApp", path: "sso-callback" }).

Browser Lifecycle Bridge: Call WebBrowser.maybeCompleteAuthSession() at the global file scope level to ensure the native OS catches returning OAuth payloads cleanly.

2. Component Logic & State Constraints
A. Authentication & Lifecycle Initialization
Import useAuth and useSSO from @clerk/clerk-expo.

Import useRouter from expo-router.

Explicitly manage a local boolean state hook const [loading, setLoading] = useState(false);.

Pull isLoaded strictly from useAuth(). Pull startSSOFlow from useSSO().

B. Navigation & Interaction Handlers
handleBack Function:

Trigger Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light).

Redirect securely back to the index screen (router.replace('/')).

handleGoogleLogic Function:

Must be an async function.

Guard Clauses: Immediately exit execution if !isLoaded or if loading === true.

Change state: Set loading to true.

Trigger tactile feedback: Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium).

SSO Strategy: Execute startSSOFlow passing strategy: "oauth_google" and the string evaluation from getClerkSSORedirectURL().

Session Registration: If createdSessionId and setActive resolve successfully, invoke await setActive({ session: createdSessionId }) before executing router.replace('/'). Wrap the sequence in a try/catch block, resetting loading to false in the finally block.
